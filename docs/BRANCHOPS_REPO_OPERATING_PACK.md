# BranchOps Repo Operating Pack

This document defines the required Codex operating rules for this repository.

## Purpose

Make every Codex job traceable, reviewable, and connected to BranchOps source-of-truth systems.

## Required Completion Protocol

Every task must end with:

1. Objective
2. Classification
3. Primary Agency
4. Departments / Roles Involved
5. Work Completed
6. Files Changed
7. Validation Performed
8. Risks / Compliance Notes
9. Source-of-Truth Updates Needed
10. Executive Review Board Decision
11. Handoff
12. Next Executable Codex Prompt

## Decision Logic

Use exactly one decision:

- APPROVE: complete and validated
- REVISE: useful but needs fixes
- BLOCK: should not proceed due to risk or bad direction
- ESCALATE: needs founder, legal, tax, security, or outside review

## Source-of-Truth Rules

- GitHub

## Standard Codex Prompt

Use this when starting repo work:

Review this repository under the BranchOps Codex Agency Operating System. Classify the task, route it through the correct departments, complete the work, validate it, identify risks, produce a source-of-truth update list, make an Executive Review Board decision, create a handoff, and end with the next executable Codex prompt.
