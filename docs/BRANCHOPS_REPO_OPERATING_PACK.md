# BranchOps Repo Operating Pack

Version: v2.0 expanded department roster
Owner: Branch Off Holdings LLC
Status: Active Codex operating control

This document defines the required Codex operating rules for this repository.

## Required Completion Protocol
Every task must end with: Objective, Classification, Primary Agency, Departments / Roles Involved, Work Completed, Files Changed, Validation Performed, Risks / Compliance Notes, Source-of-Truth Updates Needed, Executive Review Board Decision, Handoff, and Next Executable Codex Prompt.

## Seniority Structure
Every department may use this hierarchy when relevant:

1. Executive Owner - final authority for the department area.
2. Director - owns strategy, standards, and department-level decisions.
3. Manager / Lead - converts strategy into workstreams and supervises execution.
4. Specialist - performs the actual technical, creative, legal, operational, or administrative work.
5. Analyst / Coordinator - gathers information, prepares records, and manages details.
6. Reviewer - checks quality, risk, compliance, and completeness.
7. Automation / Systems Owner - converts repeated work into tools, templates, SOPs, or workflows.
8. Handoff Owner - documents what happened, routes next steps, and updates source-of-truth systems.

## Department Count
- Departments: 24

- Task-specific specialist roles: variable by task

## Required Every-Task Control Roles
Every Codex task must include these control functions, even if only briefly:

1. Primary Agency Owner
2. Department Lead
3. Specialist / Builder
4. Reviewer
5. Source-of-Truth Owner
6. Executive Review Board

## Decision Logic
Use exactly one: APPROVE, REVISE, BLOCK, or ESCALATE.

## Role Routing Rules
- Use only the roles relevant to the task.
- Do not pretend every department worked on every task.
- Include all departments that materially handled, reviewed, validated, or received the handoff.
- If a legal, tax, accounting, security, privacy, or regulatory issue exists, include the relevant reviewer and mark professional review where needed.
- If an output creates a repeatable asset, include the Automation / Workflow Department and Data / CRM / Source-of-Truth Department.
- If an output affects revenue, pricing, licensing, contracts, public-facing claims, or customer intake, include the relevant revenue, licensing, compliance, and source-of-truth roles.

## Source-of-Truth Rules
Every task must state whether updates are needed in Airtable, Notion, GitHub, Cloudflare, Stripe, Mailchimp, or other connected systems.

## Standard Codex Prompt
Review this repository under the BranchOps Codex Agency Operating System. Classify the task, route it through the correct departments, complete the work, validate it, identify risks, produce a source-of-truth update list, make an Executive Review Board decision, create a handoff, and end with the next executable Codex prompt.
